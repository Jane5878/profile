01 專案概述 Project Overview
背景與需求 Background

發放電子優惠券是公司最受歡迎的行銷工具。針對企業客戶在檔期活動中面臨的「發放成本高」、「數據追蹤斷裂」與「核銷流程繁瑣」等痛點，提出開發了此整合 LINE 生態圈的票券模組。協助 B 端客戶能自主、彈性地設計活動，並透過數位足跡落實精準行銷；也使公司開發資源能更有效的利用

核心目標 Goals
  • →刺激衝動消費：透過多元渠道（LINE/FB/EDM）激發領券意願。
  • →降低行銷成本：取代紙本券，實現高互動且低成本的數位擴散。
  • →線上線下整合：同步整合 OMO 數據，掌握從領券到店核銷的完整路徑。

03 功能模組 Modules
功能模組
  • →多渠道發送：透過LINE OA、FB、EDM發券給潛在及現有會員。
  • →動態行為貼標：根據用戶領券與點擊行為，即時更新會員標籤。
  • →OMO核銷中心：線上領券，線下門市掃碼核銷。
  • →數據儀表板：視覺化分析領券量、使用率與轉換率。

06 挑戰與學習 Challenges & Key Learnings
🔗
跨渠道身份識別
確保用戶從 FB 或 SMS 導流至 LINE 時，數位足跡能精準整合至同一會員 ID。
→ 強化了對 OpenID 與 CRM 連結邏輯的理解。
⚙️
高併發核銷穩定性
在特定檔期面臨瞬間高流量核銷壓力，需優化 API 響應速度以避免結帳塞車。
→ 引入非同步排隊機制與緩存策略優化。
💡
業務與技術的平衡
初期專注功能齊全，後續發現客戶更在意「Sales Kit」與「ROI 報表」的直覺度。
→ 調整產品方向，將數據視覺化作為核心亮點。
