01 專案概述 Project Overview
背景 Background

一款主打在地社群互動的行動應用程式，核心功能圍繞「快問快答（Q&A Story）」與社群貼文，並搭配虛擬幣（M幣）獎勵機制驅動用戶參與。技術架構採 WebView 與原生頁面混合，兼顧跨平台開發效率。

核心目標 Goals
  • →建立在地社群互動平台，提升 DAU
  • →透過 M幣獎勵生態驅動 UGC 內容量成長
  • →設計多帳號角色架構支援 B2C 品牌入駐
技術背景 Tech Context

Native 負責效能敏感模組（相機、定位等），WebView 負責內容展示。在開發成本與 UX 流暢度間取捨，體驗一致性為主要挑戰。

02 我的角色與職責 My Role & Responsibilities
功能規格定義
撰寫各功能模組 PRD，定義 User Story、測試驗收腳本，協調前後端工程對齊規格。
Roadmap 規劃
負責 Backlog 優先排序，制定版本 Roadmap，管理 Sprint 計劃與需求變更對排程的衝擊。
跨組溝通協調
同步對接前後端、UI/UX 設計、運營與行銷、業務等團隊，擔任各方溝通橋樑。
用戶研究與分析
彙整In-APP回饋與App Store Review，與 UX共同研究回饋與痛點，並依照頻率與嚴重度進行優先排序、提出改善方案，驅動功能迭代。
A/B Testing 執行
建立 A/B Testing 實驗，定義實驗對象、實驗組與對照組，並追蹤實驗數據，進行實驗結果分析，提出改善方案，驅動功能迭代。
行銷活動聯動
結合七夕行銷活動需求，定義匿名配對聊天功能規格，協調產品與行銷團隊的時程對齊與資源分配。

04 挑戰與學習 Challenges & Key Learnings
⚙️
Hybrid App 架構取捨
WebView 切換影響轉場體驗，需持續在開發成本與 UX 之間進行協商。
→ 學習技術選型對體驗決策的影響
🔬
功能分階落地
推播演算法未能及時上線，後續使用評分作為 Fallback 方案。
→ 強化 MVP 優先思維
🤝
利害關係人對齊
同步對接四個團隊，需要在緊縮的時程下保持一致的規格認知。
→ 建立高效溝通節奏
